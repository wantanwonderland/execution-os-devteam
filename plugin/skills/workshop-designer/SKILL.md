## Core Philosophy: Facilitate Discovery, Don't Deliver Content

A facilitator guides participants to discover the answer. A speaker tells them the answer. **{{OWNER_NAME}} is a facilitator, not a presenter.**

The test: If you can swap {{OWNER_NAME}} out for a YouTube video and nothing changes — the design has failed.

#### The 7-Phase Flow Architecture

Every workshop, regardless of length, follows this phase sequence. Scale timing proportionally.

| Phase | Purpose | Capstone Example | % of Time |
|-------|---------|-----------------|-----------|
| **1. HOOK** | Create gap + energy. Silence, then provocative line. Never open with "Welcome to..." | Title slide: 3 sec silence → "Everyone uses AI. Almost nobody uses it well." | 2-5% |
| **2. PRIME** | Icebreaker + credentials + alarming stat. Build urgency. Show why THIS matters NOW. | Escalation hands-up → 78% McKinsey stat → "I teach what I live" | 10-15% |
| **3. ORIENT** | State 3 measurable LOs. Participants know exactly what they'll be able to DO. | "By the end, you'll be able to: explain, identify, create." | 3-5% |
| **4. DISCOVER** | Key points via question → framework → activity cycle. Each KP opens with a question, never a statement. Participants discover the concept before {{OWNER_NAME}} names it. | KP1: "What tasks did you do yesterday?" → Two Buckets → Think+Share+Harvest. KP2: "Which type are you?" → 3 Types → Pulse check. | 35-45% |
| **5. APPLY** | Framework introduction + live demo + hands-on build. Participants create something real using THEIR work. | KP3: PCTGG elicited → Stack live example (45min→5min) → "Your Turn: build your meta-prompt" | 20-30% |
| **6. VERIFY** | Review all LOs explicitly. Test retention with recall interactions, not just "any questions?" | Review slide: check 3 LOs → 2 PCTGG recall tests → "Three for three" | 5-8% |
| **7. ANCHOR** | Body gesture + anchor phrase + whisper closer. Last impression = strongest impression. | One finger held up → "One framework. One task." → "A year from now... uncatchable." → Q&A with fallback. | 5-8% |

#### The Carry-Forward Thread (Most Critical Pattern)

**One thread from the participant's own work must carry through the entire session.** This is what makes the workshop experiential, not theoretical.

```
DISCOVER phase: Participant identifies their own repetitive task
     ↓ (carries forward)
DISCOVER phase: Participant diagnoses which AI User Type they are for THAT task
     ↓ (carries forward)
APPLY phase: Participant builds a meta-prompt for THAT SAME task
     ↓ (carries forward)
ANCHOR phase: "Tonight, paste this structure. Run it." — actionable with THEIR task
```

**Design rule**: If a participant can complete the workshop using someone else's example instead of their own — the carry-forward thread is broken. Fix it.

#### The Interaction Density Rule

The capstone achieves **one interaction every 45-60 seconds**. Interactions cycle through these types:

```
Question to room → Cold-call (with RESCUE) → Pair share/open discussion
→ Harvest (repeat back) → Show of hands → Say-along → Individual coaching
→ Silent reflection → Debrief question
```

**Minimum interaction counts per 15 minutes of content:**

| Interaction Type | Min Count | Purpose |
|-----------------|-----------|---------|
| Questions to room | 6-8 | Opens every section, breaks every monologue |
| Cold-calls (named) | 3-4 | Creates accountability, personal connection |
| Pair share / open discussion | 2-3 | Processing time, peer validation |
| Harvest (repeat back) | 2-3 | Validates participation, models active listening |
| Show of hands | 1-2 | Low-bar pulse check, {{OWNER_NAME}} raises first |
| Say-along | 1 | Embeds key framework through speaking |
| Individual coaching | 1 | Room learns by watching one person coached |
| Silent reflection | 1-2 | 5-10 seconds, {{OWNER_NAME}} stays visible and waits |

Scale proportionally for longer sessions. A 60-min workshop = 4x these counts.

#### The Debrief Pattern (Non-Negotiable)

Every activity MUST end with an explicit debrief. The debrief follows this formula:

> **"So what just happened? You [did X]. That's [principle/step/insight]."**

Capstone examples:
- After Think+Share+Harvest: *"So what just happened? You identified real tasks from YOUR work that AI could handle. That's the first step — awareness."*
- After Self-Assessment: *"Notice — nobody said Type 1 for everything. That's the honest starting point."*
- After Build Meta-Prompt: *"You just built a reusable prompt structure in 60 seconds. That's the difference between an Augmenter and an Avoider."*

**If you skip the debrief, the activity becomes entertainment, not learning.**

#### The Bridge Pattern

Every section ends with a bridge question that creates a gap for the next section:

| From | Bridge | To |
|------|--------|----|
| KP1 (Tasks not Jobs) | "The question isn't IF AI is coming. It's — what do you do about it?" | KP2 (3 Types) |
| KP2 (3 Types) | "I'm going to give you a tool that moves you to Type 1. Five letters." | KP3 (PCTGG) |
| Activity | "Notice — nobody said Type 1 for everything." | Framework |
| Framework | "But here's where it gets interesting..." | The Stack |

**Design rule**: Never transition with "OK, moving on..." or "Next, let's talk about..." — always bridge with a question or teaser that pulls them forward.

#### Escalation Icebreaker Pattern

The capstone's icebreaker uses **progressive elimination** — not a flat question:

```
"Hands up — who uses [topic]?"           → Everyone raises
"Keep your hand up if [harder filter]"    → Some drop
"[Even harder]..."                        → More drop
"[Hardest]..."                            → 1-2 remain
→ Spotlight the survivor: "Name them!"    → Creates energy + hero moment
→ Pivot to gap: "But are you getting consistently GOOD results?"
```

This pattern works for ANY topic. Replace the filter sequence. Use this instead of flat icebreaker questions.

#### The Review Architecture

Don't just list LOs at the end. **Test them with recall interactions:**

1. **State each LO as a question**: "Can you explain why AI replaces tasks, not jobs?"
2. **Cold-call for recall**: "Name the 5 letters" (tests memory)
3. **Cold-call for comprehension**: "What does each one stand for?" (tests understanding)
4. **Different person for each recall** — distributes participation
5. **Celebrate**: "Three for three." / "All five. Nice."

This turns the review from passive summary into active Kirkpatrick L2 assessment.

#### Applying the Blueprint to Different Durations

| Duration | Phases | Adaptation |
|----------|--------|------------|
| **15 min** (micro-workshop) | All 7 phases, compressed. 1 KP in DISCOVER, 1 activity in APPLY. | The capstone itself. |
| **45 min** (webinar) | All 7 phases. 2-3 KPs in DISCOVER, each with its own activity. 1 hands-on build in APPLY. | Add a second carry-forward thread if 3 KPs. |
| **90 min** (half-day module) | All 7 phases. 3 KPs with pair shares + harvests. 2 hands-on builds. Mid-session energiser. | Break into 2 blocks with a 10-min break. Each block has its own mini-VERIFY. |
| **3-4 hours** (half-day workshop) | Repeat DISCOVER→APPLY cycle 2-3 times. Each cycle has its own carry-forward thread. Bookend with shared HOOK, ORIENT, VERIFY, ANCHOR. | Add team exercises. Real deliverables produced. |
| **Full day (6-8 hours)** | 3-4 complete DISCOVER→APPLY cycles. Morning = concept + framework. Afternoon = application + creation. End with showcase/presentations. | Add Kirkpatrick L3 commitments. Post-workshop follow-up designed in. |

## Delivery Ratios

| Element | Workshop/Training | Webinar | Speaking Event |
|---------|------------------|---------|----------------|
| {{OWNER_NAME}} talking | **40%** | 50% | 60% |
| Participants doing | **45%** | 35% | 25% |
| Transitions | 15% | 15% | 15% |

**Hard rule**: No monologue longer than 45 seconds (workshop) or 60 seconds (speaking) without a question, pause, or interaction point.

## 5 Delivery Markers

Embed these in ALL speaker notes. They are non-negotiable.

| Marker | Purpose | Usage |
|--------|---------|-------|
| `[MOVE]` | Position changes | Centre for key messages. Sides during activities. Walk through room during pair shares. Step forward for closers. Step back to give space. |
| `[PAUSE]` | Silence — the most powerful tool | 2-3 sec after questions (give time to think). 3-5 sec after punchlines (let it land). Never fill silence with "so..." or "okay...". |
| `[VOICE]` | Volume and pace modulation | Drop to near-whisper for emotional moments. Firm and direct for CTAs. Measured/slow for data/statistics. Conversational for transitions. |
| `[SCAN]` | Eye contact pattern | Triangle: left → centre → right before cold calls. Lock eyes with one person during a key statement. Sweep the room after asking a question. |
| `[RESCUE]` | Fallback for dead air | Every cold-call needs a simpler backup question. Every show-of-hands needs a "or maybe..." rephrasing. Never leave participants stranded. |

## Facilitation Techniques (Use in This Order of Preference)

1. **Ask, then confirm** — Pose a question. Let someone answer. Then anchor with the framework. ("What do you think P stands for?" → [answer] → "Right — Persona.")
2. **Pair share** — "Turn to the person next to you. 30 seconds. Go." Use liberally — minimum once per key point.
3. **Harvest** — After pair share, cold-call 2-3 people to share with the room. Repeat their answer back to validate.
4. **Show of hands** — Quick engagement check. Low bar for participation. Raise your own hand first to give permission.
5. **Room say-along** — "Everyone say it with me: P-C-T-G-G." Embeds through speaking. Use for frameworks, acronyms, key phrases.
6. **Elicit before tell** — Never explain what participants can discover themselves. Ask first, then fill gaps.
7. **Individual coaching** — Walk to one person. Ask them to talk through their work. Coach in real time. The room learns by watching.
8. **Silent reflection** — "Take 15 seconds. Think about this." Then break the silence with a pair share or harvest.

## Anti-Patterns (NEVER Do These)

| Anti-Pattern | What to Do Instead |
|-------------|-------------------|
| Open a key point with a statement | Open with a question to the room |
| Monologue for 60+ seconds | Break with a question, pause, or interaction |
| Explain what participants can discover | Elicit first, confirm second |
| Stand at the front for the whole session | Move. Walk through. Get close to participants. |
| Cold-call without escape routes | Always have a [RESCUE] — a simpler version of the question |
| Fill silence with "so..." or "okay..." | Let silence do the work. Count to 3 in your head. |
| Read slides aloud | Point to the slide, then face the room and elaborate |
| Use polished TED-talk cadences | Conversational tone. Talk TO people, not AT them. |
| End with "any questions?" as the final moment | End with a body gesture, commitment, or anchor statement |

## Design Frameworks (TTT HRDF)

When designing workshops, apply ALL of these. Tag each module/slide with the frameworks it addresses.

### Knowles' 6 Principles of Andragogy

| Principle | How to Apply |
|-----------|-------------|
| **Need to Know** | Open with WHY before WHAT. Show the gap. Make WIIFM (What's In It For Me) explicit. |
| **Self-Concept** | Treat participants as peers. Honour their experience. Never lecture down. |
| **Experience** | Use THEIR work, THEIR examples, THEIR scenarios — not generic case studies. |
| **Readiness** | Create urgency. Show the problem is real and immediate. |
| **Problem-Oriented** | Every exercise solves a real problem the participant faces today. |
| **Intrinsic Motivation** | Frame choices, not mandates. "You choose your path" — internal driver. |

### Thorndike's 6 Laws of Learning

| Law | How to Apply |
|-----|-------------|
| **Readiness** | Pre-work or icebreaker creates energy + curiosity before content. |
| **Primacy** | First content = strongest message. Get it right. |
| **Exercise** | Practice > theory. Minimum 70% hands-on. |
| **Effect** | Create satisfying "first win" moments — visible, immediate results. |
| **Intensity** | Vivid, emotional, real. Statistics that shock. Stories that resonate. |
| **Recency** | Strong close = lasting impression. Body gesture, commitment ceremony, anchor phrase. |

### Bloom's Taxonomy Distribution

| Level | Target % | Activities |
|-------|----------|-----------|
| Create (L6) | ≥15% | Build prompts, design plans, generate deliverables |
| Evaluate (L5) | ≥15% | Assess own work, score readiness, critique approaches |
| Analyse (L4) | ≥15% | Compare frameworks, identify gaps, break down problems |
| Apply (L3) | ≥25% | Hands-on exercises with real scenarios |
| Understand (L2) | ≤15% | Framework explanations (brief, then practice) |
| Remember (L1) | 0% | No rote memorization — all experiential |

### Honey & Mumford Learning Styles

Every module must serve at least 2 styles:

| Style | How to Serve |
|-------|-----------  |
| **Activists** | Hands-on exercises, live tools, "do it now" moments |
| **Reflectors** | Pair discussions, "what would you change?", reflection prompts |
| **Theorists** | Framework deep-dives, models, structured systems |
| **Pragmatists** | Real work scenarios, their own documents, immediate application |

### Serial Position Curve

| Position | Design Rule |
|----------|------------|
| **Opening (Primacy)** | Strong hook. 3-stack technique: Shared Experience + Alarming Fact + Body Gesture. Or open with a question that creates a gap. |
| **Middle (Attention dip)** | Activity every 15-20 min. No lecture >15 min. Pair shares and exercises combat the dip. |
| **Closing (Recency)** | Body gesture + anchor phrase. Public commitment. The last thing they remember is their own commitment or a vivid image. |

### Kirkpatrick Evaluation (Build In, Don't Bolt On)

| Level | Embed During Design |
|-------|--------------------|
| **L1 Reaction** | End-of-session pulse: "Rate this 1-5 on usefulness" |
| **L2 Learning** | Review slide: "Can you [LO1]? [LO2]? [LO3]?" — elicit from room |
| **L3 Behaviour** | 30/60/90 day check-ins. "What task did you augment this month?" |
| **L4 Results** | Post-workshop impact survey (Google Form). Tie to business KPIs. |
