# Code Ownership Workflow

When Levi is dispatched for code ownership analysis, follow this workflow.

## Input

Levi receives an optional directory scope. If no scope is provided, analyze the entire repository.
Default lookback window: 90 days.

---

## Phase 1: Gather Commit History

For each directory in scope, count commits per author over the last 90 days:

```bash
# All top-level directories
for dir in $(git ls-tree --name-only HEAD); do
  echo "=== $dir ===";
  git log --since="90 days ago" --format="%aN" -- "$dir" | sort | uniq -c | sort -rn;
done

# Single directory scope
git log --since="90 days ago" --format="%aN" -- {dir} | sort | uniq -c | sort -rn
```

Also collect the full list of active staff from `data/company.db`:

```sql
SELECT name, github_handle FROM staff WHERE status = 'active';
```

Map git author names to `staff` table entries. If a git author name does not match any active staff record, flag it as an unknown author (may be a former employee or contractor).

---

## Phase 2: Bus Factor Analysis

For each directory, calculate the ownership concentration:

```
total_commits = sum of all author commit counts for the directory
top_author_pct = (top_author_commits / total_commits) * 100
```

**Bus factor = 1** when a single author has >80% of commits.

Flag these directories with severity:

| Condition | Severity |
|-----------|----------|
| Top author >80% of commits, 0 other active contributors | CRITICAL — single point of failure |
| Top author >80% of commits, 1 other active contributor | HIGH |
| Top author 60-80% of commits | MEDIUM — concentration risk |
| Top author <60% of commits | OK |

Also flag directories where the top contributor is no longer active staff (departed employee owns critical module).

---

## Phase 3: Knowledge Gap Analysis

A knowledge gap is a module where no currently active staff has committed in the last 90 days.

```bash
# Get all authors for a directory in last 90 days
git log --since="90 days ago" --format="%aN" -- {dir} | sort -u
```

Cross-reference against active staff list:
- If zero matches → **Knowledge Gap: No active staff has touched this module recently**
- If only 1 match → **At Risk: Single active contributor**

Also flag modules with high commit frequency but low contributor diversity (many commits, all from one person):
- "High activity, low diversity" — important module, high bus factor

---

## Phase 4: Generate CODEOWNERS

Assign ownership based on the top contributor per directory over the last 90 days. If top contributor is not active staff, assign the second-highest active contributor. If no active staff contributor exists, leave unassigned and flag.

CODEOWNERS format (GitHub standard):

```
# CODEOWNERS — generated by Levi on {date}
# Based on git commit history (last 90 days)
# Review and adjust: automated assignment is a starting point, not a final decision

# Global fallback owner (optional — set to tech lead)
# * @{tech-lead-github-handle}

# Source directories
/src/auth/          @{github_handle}
/src/payments/      @{github_handle}
/src/api/           @{github_handle}
/src/frontend/      @{github_handle}

# Infrastructure
/.github/           @{github_handle}
/terraform/         @{github_handle}
/docker/            @{github_handle}

# Documentation
/docs/              @{github_handle}
```

Rules for CODEOWNERS generation:
1. Use `github_handle` from `staff` table, prefixed with `@`
2. If a directory has no active contributor, add a comment: `# UNOWNED — no active contributor in last 90 days`
3. Subdirectories with a different dominant owner than their parent get their own entry
4. Sort: global fallback first, then source dirs, then infra, then docs

---

## Phase 5: Compile Reports

### Bus Factor Report

```markdown
## Bus Factor Report — {date}

**Repo**: {repo} | **Analysis window**: 90 days | **Scope**: {all / dir}

### CRITICAL — Single Points of Failure

| Directory | Top Author | Ownership % | Other Active Contributors |
|-----------|------------|-------------|--------------------------|
| /src/auth | alice | 94% | 0 |

**Risk**: If alice is unavailable, the auth module has no backup owner.
**Recommendation**: Pair-program or knowledge transfer sessions for these modules.

### HIGH — Concentration Risk

| Directory | Top Author | Ownership % | Other Active Contributors |
|-----------|------------|-------------|--------------------------|
| /src/payments | bob | 83% | 1 (charlie — 4 commits) |

### MEDIUM

| Directory | Top Author | Ownership % | Other Active Contributors |
|-----------|------------|-------------|--------------------------|
| /src/api | alice | 67% | 2 |

### OK (well-distributed)

{count} directories have healthy ownership distribution (top author <60%).
```

### Knowledge Gap Report

```markdown
## Knowledge Gap Report — {date}

**Repo**: {repo} | **Analysis window**: 90 days

### Gaps (zero active staff commits)

| Directory | Last Commit Date | Last Active Author | Status |
|-----------|-----------------|-------------------|--------|
| /src/legacy-importer | 2024-08-15 | dave (departed) | ORPHANED |
| /scripts/migrations | 2025-01-20 | (unknown) | UNOWNED |

**Risk**: These modules have no active owner. Any incident here has no clear responder.

### At Risk (single active contributor)

| Directory | Active Contributor | Last Commit |
|-----------|--------------------|-------------|
| /src/integrations/stripe | alice | 2025-03-28 |

### Recommendations

1. Assign knowledge transfer sessions for CRITICAL and ORPHANED modules
2. Add unowned directories to the tech debt backlog
3. Consider deprecating orphaned modules with no business value
```

---

## Phase 6: Output

1. **CODEOWNERS file**: Write to `{repo-root}/CODEOWNERS` — overwrite if exists, otherwise create
2. **Bus factor report**: Write to `09-ops/ownership/YYYY-MM-DD-bus-factor.md`
3. **Knowledge gap report**: Write to `09-ops/ownership/YYYY-MM-DD-knowledge-gaps.md`
4. **DB**: Insert summary row into `tech_debt` table for each CRITICAL finding:
   ```sql
   INSERT INTO tech_debt (repo, title, severity, category, created_at, owner)
   VALUES ('{repo}', 'Bus factor: {dir} owned by single contributor ({name})', 'high', 'documentation', datetime('now'), '{name}');
   ```

---

## Constraints

- NEVER write a CODEOWNERS file without first showing the bus factor and knowledge gap reports
- Automated assignments are starting points — always include a note to review
- Departed staff must never appear in CODEOWNERS — only active staff
- If `data/company.db` is unavailable, proceed with git history only and flag that staff cross-reference was skipped
- Log analysis to wantan-mem as observation for trend tracking
