Sprint retrospective. Identify patterns and improvement actions.

## Steps

1. **Data gathering**: Dispatch Erwin to analyze this sprint's data:
   - Standup logs from `06-ceremonies/standup/` (blocker patterns, recurring themes)
   - Incident data (root causes, response times)
   - PR review data (bottlenecks, SLA violations)
   - Deploy data (failures, rollbacks)

2. **Pattern detection**: Present top 3 patterns found in the data. For each: what happened, how often, impact.

3. **Stop/Start/Continue**: Ask the team:
   - **Stop**: "What should we stop doing?"
   - **Start**: "What should we start doing?"
   - **Continue**: "What's working well that we should keep?"

4. **Action items**: For each improvement, create a concrete action with owner and due date. Write as tasks with `type: task` in `06-ceremonies/retro/`.

5. **Write retro**: Create `06-ceremonies/retro/YYYY-MM-DD-sprint-{id}-retro.md` with patterns, stop/start/continue, and action items.

6. **Check previous retro actions**: Read the last retro file. For each action item, ask: "Was this completed?" Update status.
